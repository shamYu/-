性能和集成

Web Worker

Web Workers 是 HTML5 提供的一个javascript多线程解决方案，我们可以将一些大计算量的代码交由web Worker运行而不冻结用户界面。 Web Worker的基本原理就是在当前javascript的主线程中，使用Worker类加载一个javascript文件来开辟一个新的线程，起到互不阻塞执行的效果，并且提供主线程和新线程之间数据交换的接口：postMessage，onmessage。

//worker.js
onmessage =function (evt){
  var d = evt.data;//通过evt.data获得发送来的数据
  postMessage( d );//将获取到的数据发送会主线程
}

//页面上交互
var worker =new Worker("worker.js"); //创建一个Worker对象并向它传递将在新线程中执行的脚本的URL
worker.postMessage("hello world");     //向worker发送数据
worker.onmessage =function(evt){     //接收worker传过来的数据函数
    console.log(evt.data);           //输出worker发送来的数据
}

web worker 开启了一个新的线程去进行一些复杂的耗时的操作，我们可以做什么：

  1. 可以加载一个JS进行大量的复杂计算而不挂起主进程，并通过postMessage，onmessage进行通信
  2. 可以在worker中通过importScripts(url)加载另外的脚本文件
  3. 可以使用 setTimeout(), clearTimeout(), setInterval(), and clearInterval()
  4. 可以使用XMLHttpRequest来发送请求 5.可以访问navigator的部分属性

有那些局限性：

1.不能跨域加载JS 2.worker内代码不能访问DOM 3.各个浏览器对Worker的实现不大一致，例如FF里允许worker中创建新的worker,而Chrome中就不行 4.不是每个浏览器都支持这个新特性

requestAnimationFrame

关于动画， 我们都知道渲染频率要达到 60fps 才能有流畅细腻的动画，在之前我们都用 setTimeout* 或是 setInterval** 去完成 60fps 的设定， 这样的做法，在正常情况下没有任何问题，但是一旦设备cpu速度不够，或是页面脚本繁忙的时候，根本达不到 60 fps，造成页面卡顿。

HTML5 给我们带了 requestAnimationFrame， 它会在页面重绘（页面重绘的频率也是 60fps）前，通知浏览器调用一个指定的函数，以满足开发者操作动画的需求。这和方法接收一个函数为参数，该函数会在重绘前调用。

这了更好的兼容不支持 requestAnimationFrame 的浏览器，我需要写一些 polyfill 的方法:

window.requestAnimationFrame = (function() {
    return window.requestAnimationFrame ||
          window.webkitRequestAnimationFrame ||
          window.mozRequestAnimationFrame ||
          window.oRequestAnimationFrame ||
          window.msRequestAnimationFrame ||
          function(callback) {
              return window.setTimeout(callback, 1000 / 60);
          };
})();

dom 查询api

dom 查询一直是 javascript 的痛点，之前只有 getElementById , getElementsByName() , getElementsByTagName 这三个API来查询dom 非常的不方便， 之所以如此 JQuery 问世的时候，才如此的火热，它告诉了W3C改这么做dom 操作的额api, 于是乎，在html5中加入了这几个方法

querySelector() querySelectorAll()

这个两个方法和jquery一样，是以 css selector 的方式来查询dom的。 犹豫是原生操作，它的查询速度会比jquery好很多。并且在ie8以上浏览器中都得到了支持.

<input type="text" name="name" value="">

<script type="text/javascript">
    $("input[name=name]")[0] === document.querySelector("input[name=name]"); //true
</script>
