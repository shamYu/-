移动执法3.0开发文档

插件开发

render也提供插件开发机制，以插件的方式扩展本身的功能，主要以拦截器器，$.fn.render 内部方法调用拓展。

拦截器

之前有讲过render在页面初始化时会扫描元素，调用组件渲染，那么在某个组件渲染之前想做些操作该怎么做呢？拦截器就是为了解决这个问题

拦截器的定义

render.addInterceptor({
    onSingleElementRenderSuccess: function(element, component){
        //单个元素渲染成功时（不包括子元素）
        //element 元素， component 组件对象
    },
    onElementDestory: function(element, name){
        //单个元素卸载时
        //element 元素， name 组件名称
    },
    onBeforeElementRender: function(element){
        //在元素渲染之前
    },
    onElementRenderSuccess: function(element){
        //在元素渲染及其子元素渲染完成后执行
    },
    onDestory: function(element){
        //卸载时
        //element 元素
    }
});

扩展拦截器

插件内部也可以扩展一些拦截器，

var TEST_INTERCEPTOR = 'TEST_INTERCEPTOR'; //常量名要大写

//在某处执行所定义的此拦截器
render.doInterceptor(TEST_INTERCEPTOR, 'test params');

render.doInterceptor(name, [arg1, arg2]) 执行定义的拦截器 name 拦截器名车， arg传递的参数

$.fn.render 方法扩展

$.fn.render 内置了一些方法，以 $(selector).render('') 方式调用


$(selector).render('get');   //获取元素组件对象
$(selector).render('destory'); // 销毁元素
$(selector).render('render', 'iselect', {}) //渲染单个元素

不同的插件也可以扩展这些方法

render.jqMethod('testAdd', function(msg){
    console.log('this is test ' + msg);
});

//调用
$(selector).render('testAdd', '测试而已');

// this is test 测试而已

日志处理

render中的日志主要基于 console 对象开发，有 log、info、warn、error 四个级别的日志，输出级别可以自由配置

render.config.log = true; //输出所有级别的日志
render.config.log = false; //不输出日志
render.config.log = 'error'; //只输出 error级别的日志
render.config.log = ['warn', 'error']; //只输入warn、error级别的日志

日志方法调用

var log = render.log;
log.info('');
log.log('');
log.warn('');
log.erro('');

合理的日志打印，可以在开发的时候给予很大的帮助。

插件定义

render.install('factory', function () {
    var map = {};
    return function (name, fun) {
        if (!name) return null;
        if (name && !fun) return map[name];
        var obj = fun;
        if ($.isFunction(fun)) obj = fun();
            return (map[name] = obj);
    };
});

//现在render.factory 就等于 function内 return 回来的function.
//这里可以return很多例如function， object， 字符串。。。。
