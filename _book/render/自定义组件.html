移动执法3.0开发文档

组定义组件

render 在页面初始化时会自动扫描每一个元素，当发现元素上带有 render 属性时，会找到相应的组件对其进行渲染。所以对于页面的渲染逻辑是卸载组件定义中。render 对于组件的定义我是借鉴 jqueryui 的 widget 思想，提供完善的 组件继承和混合方式，还是有参数、事件等等处理。

最简单的组件

定义一个hello 按钮的组件， 当点击按钮时会在按钮之前显示hello world

//组件名称hello
$.render('hello', {
    options: {
        msg: 'hello word'
    },
    createDom: function(element, options){
        return $('<span>' + options.msg + '</span>').insertAfter(element)[0];
    },
    init: function(){
        var self = this;
        //this.$target 代表 render标记的元素JQuery对象
        this.$target.on('click', function(){
            self.$element.toggle(); //$element createDom返回元素的JQuery对象
        })
    },
    destory: function(){
        //当这个组件销毁之后调用次方法
    }
})

使用时跟之前一样写在标签里即可

<button render="hello" name="button"></button>
<!--自定义msg内容-->
<button render="hello" options="msg:'hello 这世界'" name="button"></button>

options

由上述的代码可以看到，options决定了组件的默认参数，如果在使用组件是，指定了参数值，就使用指定参数的值，没有则取默认值。在定义组件的时候尽量考虑好常用的设置，给予相应的默认值，让开发时尽量少的配置则可完成功能。

在组件内部访问options可以使用 this.options访问

createDom

让组件需要额外创建dom或是改变现有的dom结构是，则可以使用 createdDom ,它是一个函数，将创建或是修改dom的操作写在这里，将新的dom返回回来。

this.$element //返回的新的dom的jQuery对象
this.$target //原始dom的jQuery对象

this.targetElement //原始dom
this.element      // 返回的新的dom

init 和 destory

组件在createDom操作完成之后，会调用init方法完成初始化操作，这个是必需定义的方法，这里最好用来 绑定事件 和 初始化的数据加载

当组件被jq的 remove 方法移除时会自动调用destory方法（这里改写过jq的代码）, 如果没有一些资源需要释放的，这里可以不写. destory 里应处解绑一些与组件内dom无关的事件， 例如： window事件或是别的dom的事件。清空一些缓存的数据，这样可以节省雨多不必要的内存消耗。

触发事件

组件内部的动作，在外边是无法监听到的，所以，当组件内部发生改变时，需要触发事件，事件名以 组件名.事件名 方式命名，并保证用 targetElement 触发，这个部分方法已经封装好了。

this._trigger('event'); //这里会用targetElement触发事件，并自动加上组件名称

定义表单组件

表单组件和基础的组件有些不同

  1. 必须继承form组件
  2. 加入了更多需要配置的方法
$.render('iText', 'form', {
    options: {

    },
    init: function(){
        this._super.apply(this, arguments); //调用父组件的此方法
    },
    change: function(){
        //targetElement 值改变时调用方法
        // 这里修改过jq的 val方法，当使用这个方法改变值得时候就会调用组件的change方法 此处需要加上，修改dom逻辑
    },
    focus: function(){
        //元素获取焦点时调用此方法
    },
    blur: function(){
        //元素失去焦点时调用此方法
    },
    valid: function(valid, msg){
        //这里修改了easyui的 验证方法，当元素要验证时会调用此方法
        //valid 是否验证通过
        //msg 验证提示信息
    }

})

form表单定义模板就是上面这样了，需要注意的是， focus、blur, valid在form组件中都有做处理，非必要情况不必重写。
