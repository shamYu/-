移动执法3.0开发文档

render-module 模块化

移动执法3.0中使用了许多ajax请求页面的功能， 当一个页面整合了许多别的页面的时候，会发生许多的dom冲突， js冲突等等的问题，加上很多信息维护页面的逻辑基本一致，没必要每次都copy代码，只需要将逻辑抽象出来即可。

模块化的基本使用

简单来说，现在的这个模块化，就是讲原来卸载全局变量里的方法，写在一个构造器里，构造器不会重新加载,像这样：

<div render-module="hello: { msg: '啦啦啦啦啦'}">
    <button name="button" id="hello">hello</button>
</div>
<script type="text/javascript">
    render.module('hello', 'base', {
        params: {
            msg: 'tttt'
        },
        init: function(context, params){
            $(context).on('click', '#hello', function(){
                alert(params.msg);
            })
        }
    });
</script>

render-module是 render的一个插件， 当render在扫描时发现有 render-module属性的元素， 就会获取它的值，分析出 module的名称, 和加载传递的参数（以 name: {} 这种方式写）, 根据名称实例化一个module的构造器，以这种面向对象的方式编码，不管这个jsp在同一个页面加载多少次都不会相互冲突、

context

当某个模块被加载之后 this.context 则指向 render-module 定义的那个元素，我们要保证，module内部所有的dom操作，都只能在 this.context 下，以保证不和其他的页面冲突，像这样：

$('#hello', this.context);
//或者是这样
this.$get('#hello');

binds

binds 是一个需要绑定事件对象数组， 其中 selector 是需要被绑定事件的元素的选择器， event 事件类型， handler 可以是一个 function(){} 也可以字符串，标识模块对象内的方法。

当模块初始化的时候会自动扫描binds数据，以事件委托的方式将这些事件绑定是 context （render-module定义的元素）元素上

render.module('hello', 'base', {
    params: {
        msg: 'ttttttt'
    },
    binds: [{
        selector: '#hello',
        event: 'click',
        handler: 'showMsg'
    }],
    init: function(){
        //初始化操作
    },
    showMsg: function(){
        alert(this.params.msg);
    }
})

大家可能注意到了，这个module是可以被继承的，所以我们将很多也页面的逻辑都抽象出来，做了许多的基础模块(base, datagrid, form, list, sideList, sideMenu, subList)。
