移动执法3.0开发文档

GenericService 使用

平台封装的 serviceHelper 分页查询， sql + bindParams 生成查询语句， 通常的情况这么写:

public void queryGgdmj(HttpServletRequest request, HttpServletResponse response) throws Exception{

        ServiceHelper helper = new ServiceHelper(request,response, this.getClass(), "QUERY_GGDMJ");
        StringBuilder sql = new StringBuilder();

        sql.append(helper.bindWhere("searchDmjBhMc", "DMJBH", Types.VARCHAR,  LogicSymbol.Like, ConnectSymbol.And, BracketType.SMALL_LEFT));
        sql.append(helper.bindWhere("searchDmjBhMc", "DMJMC", Types.VARCHAR, LogicSymbol.Like,ConnectSymbol.Or, BracketType.SMALL_RIGHT));
        sql.append(helper.bindWhere("searchDmjMk", "SXMK", Types.VARCHAR, LogicSymbol.Like));

        helper.replaceSql("#WHERESTRING#",sql.toString());
        String result = helper.executePagedSQL(null, "DMJBH");
        helper.write(result);
}

<sql id="QUERY_GGDMJ" orderby="XGSJ DESC">
    <both><![CDATA[
        SELECT * FROM T_COMN_GGDMJ WHERE 1=1 AND XGSJ IS NOT NULL #WHERESTRING#
    ]]></both>
</sql>

于是，我们每一个查询的数据的方法都要写一个 query方法和sql配置， query方法实际只是执行了数据绑定到 sql的逻辑而已，我们可以吧这部分的逻辑抽象处理，成为一个公共的方法，再查询时，只用配置sql.xml， 和ajax请求不需要反复的写 query 方法。

另外，这种mvc的方式，定义的方法必须传入 HttpServletRequest 和 HttpServletResponse 作为参数，这个servlet有啥子区别呢？只不过把一个servlet类变为了许多的方法，和servlet的耦合度太高， 这个部分，我们定义了 business 方法做中间层转发， 实际的service里面不限制方法的参数传入，可以随意些，更加解耦。

SqlParam 对象

sql 绑定参数对象，根据 serviceHelp 传入的绑定参数而定的。

public class SqlParam {
    /**
     * 参数类型
     */
    public static enum Type {
        STRING,
        INTEGER,
        FLOAT,
        DATE,
    }
    private String key;
    private String value;
    private Type type;  
    private LogicSymbol logicSymbol; //比较符号
    private ConnectSymbol connectSymbol; // 链接符号
    private BracketType bracketType ; //括号定义
    public SqlParam(){}
    public SqlParam(String key, String value, Type type){
        this.key = key;
        this.value = value;
        this.type = type;
    }
    //省略了set get 方法
}

key 对应 column 即查询的列名 value 值 type 类型 STRING，INTEGER, FLOAT, DATE logicSymbol 逻辑符号 Like("LIKE"),LeftLike("LIKE"),RightLike("LIKE"),Equals("="),Greater(">"),Lesser("<"),GreaterOrEquals(">="),LesserOrEquals("<="),IsNull("IS NULL"),IsNotNull("IS NOT NULL"),NotEquals("<>"),In("IN"); connectSymbol 连接符号 And("AND"),Or("OR"),None(""); bracketType 括号定义 BIG_LEFT("{"),BIG_RIGHT("}"),MIDDLE_LEFT("["),MIDDLE_RIGHT("]"),SMALL_LEFT("("),SMALL_RIGHT(")"),NONE("");

business 方法

此方法可以让 javascript 远程调用java的方法，并且返回结果。

//截取 HandBookService 中的一段代码
public class HandBookService extends GenericService{
    //treeCatalog 方法, 接受catalogId 为参数, 返回List
    public List<Map<String, Object>> treeCatalog(String catalogId) throws IllegalAccessException, NoSuchMethodException, InvocationTargetException {
       if(StringUtils.isEmpty(catalogId)) return null;
       List<HandBook> handBookList = orm.clear().bindWhere("isCatalog", 1).bindWhere("type", FILETYPE)
               .bindWhere("catalogId", catalogId).orderBy("sortNum", Orm.SortWay.ASC).search();
       if(handBookList == null || handBookList.isEmpty()) return null;
       List<Map<String, Object>> list = new ArrayList<Map<String, Object>>();
       for (HandBook book  : handBookList) {
           Map<String, Object> map = new HashMap<String, Object>();
           map.put("id", book.getId());
           map.put("name", book.getName());
           map.put("children", treeCatalog(book.getId()));
           list.add(map);
       }
       return list;
   }
}

<nbean:ajaxUrl var="treeCatalogUrl" clazz="com.szboanda.ydzf.handbook.HandBookService" method="business" parameters="actionType=treeCatalog"/>

function loadTree(catalogId){

        dataType: 'json',
        type: 'post',
        data: {
            params: JSON.stringify([{
                value: '1',
                type: 'STRING'
            }])
        },
        success: function(data){
            data = data.result; //返回的结果
        }
    })
}
</script>

这种方式将方法和servlet完全的结构， 它可以被前端调用，也可被其他的方法调用，便于jUnit测试。需要注意一下几点

actionType 必须指定，指向方法名称. params 是参数数组 ，其中 value是值， type是类型 java 方法的参数类型不可以使用包装类型 如 Integer Float 这些， 需使用 int Float

目前可以解析的参数类型：

  • STRING {value : ’ttt’, type:’STRING’} 解析字符串类型参数
  • boolean {value : ’ttt’,type:’BOOLEAN’} 解析布尔类型参数
  • Integer {value : 2,type:’INTEGER} 解析int类型参数
  • long {value : 2,type:’LONG’} 解析long类型参数
  • double {value : 2,type:’DOUBLE’} 解析double类型参数
  • float {value : 2,type:’FLOAT} 解析float类型参数
  • Date {value: ‘1994-02-03’,,type : ‘DATE’} 解析date类型参数
  • Map {value : {aa : ‘aa’,bb : ‘bb’},type : ‘MAP’} 解析Map类型参数
  • Array {value : ‘aaa,bbb,cc’, type : ‘ARRAY’} 解析字符串数组类型参数
  • LIST {value :[ {aa : ‘aa’, bb : ‘bb’}],type : ‘LIST,className : ‘MAP’} 解析list集合类型参数 className ： 集合中的对象的类，如果是list则直接写MAP即可，其他则需要写完整类名
  • Bean {value : {,aa : ‘aa’,bb : ‘bb’},,type : ‘BEAN’className : ‘com.test’} 解析javabean类型参数 需要指定类名
